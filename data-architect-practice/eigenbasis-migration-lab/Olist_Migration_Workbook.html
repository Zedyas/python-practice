1.2 — Migrate Postgres → DuckDB (the EL of ELT) from spec

Goal: copy every source table into a DuckDB file as raw_<table>, unmodified. Use DuckDB's Postgres extension to read directly — no intermediate CSV round-trip.

Spec (write it yourself):

  • Connect to a new DuckDB file olist_target.duckdb.
  • Install + load the postgres extension, then ATTACH the Postgres DB read-only.
  • For each of the 9 tables, CREATE OR REPLACE TABLE raw_<t> AS SELECT * FROM src.<t>.
  • Print each row count as you go.
✅ Checkpoint

In DuckDB: select count(*) from raw_orders; equals the Postgres count exactly. All 9 raw_* tables present. (You'll prove faithfulness rigorously in Phase 3 — this is just the smoke test.)

💡 Nudgelevel 1
DuckDB can query Postgres live: INSTALL postgres; LOAD postgres; then ATTACH '...conn string...' AS src (TYPE postgres, READ_ONLY). After attaching, src.orders is a normal table to SELECT from.
🔑 Bigger hintlevel 2
The attach connection string is libpq style, space-separated: 'host=localhost port=5432 dbname=olist user=... password=...'. Loop over a Python list of the 9 table names and run the create or replace per table, then a select count(*) to print.
📖 Answer keyreveal
Two pieces people miss: the ATTACH ... (TYPE postgres, READ_ONLY) options clause, and reading creds from os.environ inside the f-string.
# scripts/migrate_to_target.py
import duckdb, os
con = duckdb.connect("olist_target.duckdb")
con.execute("install postgres; load postgres;")
con.execute(f"attach 'host=localhost port=5432 dbname=olist "
            f"user={os.environ['PG_USER']} password={os.environ['PG_PASS']}' "
            f"as src (type postgres, read_only)")
for t in ["customers","orders","order_items","products","sellers",
          "payments","reviews","geolocation","category_tr"]:
    con.execute(f"create or replace table raw_{t} as select * from src.{t}")
    n = con.execute(f"select count(*) from raw_{t}").fetchone()[0]
    print(f"migrated raw_{t}: {n:,}")

This is the EL of ELT: raw bytes land in the target untouched as raw_*. All transformation happens afterward, in dbt (Phase 2).

1.3 — Explain it back no code

Goal: the interview rep. In writing (3–4 sentences, out loud counts), answer: why did you load raw data into a landing zone before transforming it? Why is that "ELT" and not "ETL," and why did the modern warehouse era make ELT the default?

✅ Checkpoint

Your answer names: (1) transform happens in-warehouse after load, (2) it preserves an untouched raw copy you can re-transform, (3) cheap, scalable, co-located compute made it viable. If you hit all three without peeking, check the box.

💡 Nudgelevel 1
Think about what you lose with ETL when a business rule changes after load — and what a raw landing zone lets you do that a pre-transformed pipeline can't.
📖 Answer keyreveal
Compare your wording to this, but yours is what you'll say in the room.

Load-first means the warehouse holds an untouched raw copy; transforms run on top of it in SQL/dbt, so when a business rule changes you re-run transforms instead of re-extracting from source. That's ELT. It won because warehouse storage/compute became cheap and co-located — pushing transformation into the warehouse beats moving data out to a separate transform tier.

2.3 — Fact model: fct_orders from spec

Goal: a fact table at grain = one row per order, enriched with line-item rollups and a delivery-time measure.

Spec:

  • Start from stg_orders.
  • Roll up stg_order_items to order level: item count, sum of price, sum of freight.
  • Left join rollups onto orders (an order can have zero items captured — don't drop it).
  • Add days_to_deliver = days between purchase and delivery.
  • coalesce the money/count rollups to 0 so unmatched orders aren't null.
✅ Checkpoint

Row count of fct_orders equals stg_orders (grain preserved — the join didn't fan out or drop rows). No null items_total. days_to_deliver is null only where delivery date is null.

💡 Nudgelevel 1
Two CTEs: orders from ref('stg_orders'), and items that groups ref('stg_order_items') by order_id. Then left join ... using (order_id). The grain test is the real lesson — if the count changes, your join grain is wrong.
🔑 Bigger hintlevel 2
items CTE: select order_id, count(*) item_count, sum(price) items_total, sum(freight_value) freight_total from {{ ref('stg_order_items') }} group by 1. Delivery measure: date_diff('day', purchased_at, delivered_at) (DuckDB syntax).
📖 Answer keyreveal
Watch the coalesce(..., 0) wrappers and left join using (order_id) — that combination preserves grain while filling gaps.
with orders as (select * from {{ ref('stg_orders') }}),
items as (
    select order_id, count(*) as item_count,
           sum(price) as items_total, sum(freight_value) as freight_total
    from {{ ref('stg_order_items') }} group by 1
)
select
    o.order_id,
    o.customer_id,
    o.order_status,
    o.purchased_at,
    o.delivered_at,
    date_diff('day', o.purchased_at, o.delivered_at) as days_to_deliver,
    coalesce(i.item_count, 0)   as item_count,
    coalesce(i.items_total, 0)  as items_total,
    coalesce(i.freight_total,0) as freight_total
from orders o left join items i using (order_id)

The grain test is the real lesson: fct_orders must have the same row count as stg_orders. The left join (not inner) keeps orders with no captured items; coalesce turns their nulls into 0.

3.1 — Write the checks config from spec

Goal: a checks.yml validating the Postgres→DuckDB migration across the seven layers. Start from checks.example.yml but decide each check yourself before comparing.

Spec — one check per layer, minimum:

  • Completeness: row_count on orders and order_items.
  • Financial: aggregate sum of price and freight_value, tolerance 0.0.
  • Cardinality: distinct_count of customer_id.
  • Drift: grouped_count by order_status.
  • Content: hash_total over [order_id, customer_id, order_status].
  • Referential: orphan check, order_items → orders.
  • Quality: null_rate of order_delivered_customer_date.
✅ Checkpoint

uv run reconcile run --config checks.yml --out ../reports runs and all checks PASS against your faithful migration. (If something fails now, your Phase 1 migration wasn't clean — good catch.)

💡 Nudgelevel 1
Connections use ${ENV_VAR} interpolation so creds stay out of git. Source = postgresql+psycopg2://${PG_USER}:${PG_PASS}@localhost:5432/olist; target = duckdb:///./olist_target.duckdb. Why must the financial tolerance be exactly 0?
🔑 Bigger hintlevel 2
Each check is a YAML dict with name, type, and type-specific keys. Money requires tolerance: 0.0 — a one-cent drift across a migration is a real defect, not noise. The referential check needs child_table/child_key/parent_table/parent_key.

3.2 — Break it, then catch it from spec

Goal: the core learning rep and your best interview demo. Corrupt the target deliberately, predict which checks will fail, then run the harness and confirm.

Do this in order:

  • In olist_target.duckdb: delete all raw_orders rows where status = 'shipped'.
  • Then nudge price up 1% on ~50 orders' worth of raw_order_items.
  • Before running: write down which of your 7 checks you expect to FAIL and why.
  • Run the harness. Compare reality to your prediction.
✅ Checkpoint

row_count, orders_by_status (drift names the shipped bucket), price_sum, and content_hash all flip to FAIL. The others may still pass. Your written prediction matched the report. The reports/*.md file is your audit artifact.

💡 Nudgelevel 1
Deleting shipped rows hits: total count, the status grouped-count, and (since those orders had items) anything summing across them. The 1% price bump is sneakier — which check catches a value change that doesn't change row counts? That's the one that earns the fee.
🔑 Bigger hintlevel 2
SQL to break it: delete from raw_orders where order_status='shipped'; and update raw_order_items set price = price*1.01 where order_id in (select order_id from raw_orders limit 50);. The price bump shows up in price_sum (financial) and, if you'd hashed price, content_hash — counts alone would miss it entirely. That's the argument for layered validation.
📖 Answer keyreveal
The teaching point: a migration can pass row counts and still be corrupt. Layered checks each expose what the others let through.

Break the target (run in DuckDB against olist_target.duckdb):

-- 1. drop rows: hits count, status drift, and sums over those orders
delete from raw_orders where order_status = 'shipped';

-- 2. silent value drift: changes values WITHOUT changing row counts
update raw_order_items set price = price * 1.01
where order_id in (select order_id from raw_orders limit 50);

Expected result after rerunning the harness:

  • orders_row_count → FAIL (rows missing)
  • orders_by_status → FAIL, drift breakdown names the shipped bucket with the exact delta
  • order_items_price_sum → FAIL (the 1% bump)
  • orders_content_hash → FAIL if any hashed column changed
  • others may still PASS

The 1% price bump is the one that matters: it changes no row counts, so only the financial sum (and a content hash over price) catches it. That single fact is the whole argument for layered validation — and the line that sells the engagement. """
Eigenbasis Reconciliation Harness — core engine.

A parameterized migration-validation harness. You point it at a SOURCE and a
TARGET connection and hand it a list of checks (defined in YAML). It runs every
check against both sides, compares results, and emits a pass/fail audit report.

Design goals:
  - Source/target agnostic: any SQLAlchemy-compatible DB (Postgres, DuckDB,
    Snowflake, BigQuery via connectors). The harness only speaks SQL + DataFrames.
  - Checks are *data*, not code. New validations are YAML entries, not new
    functions. This is what makes it a sellable methodology artifact rather than
    a one-off script.
  - Deterministic, auditable output: every check produces a structured result
    row with source value, target value, delta, tolerance, and pass/fail.

This is the IP. The dbt project and the warehouse are commodity; this is the
differentiator. Keep it clean.
"""

from __future__ import annotations

import math
from dataclasses import dataclass, field
from datetime import datetime, timezone
from enum import Enum
from typing import Any

import pandas as pd
from sqlalchemy import create_engine, text
from sqlalchemy.engine import Engine


class CheckType(str, Enum):
    ROW_COUNT = "row_count"           # count(*) on both sides
    AGGREGATE = "aggregate"           # sum/avg/min/max of a column, optionally grouped
    DISTINCT_COUNT = "distinct_count" # count(distinct col)
    HASH_TOTAL = "hash_total"         # checksum across rows (order-independent)
    GROUPED_COUNT = "grouped_count"   # count(*) grouped by key — finds row-level drift
    REFERENTIAL = "referential"       # orphan check: child keys with no parent
    NULL_RATE = "null_rate"           # fraction of NULLs in a column


class Status(str, Enum):
    PASS = "PASS"
    FAIL = "FAIL"
    ERROR = "ERROR"


@dataclass
class CheckResult:
    name: str
    check_type: CheckType
    status: Status
    source_value: Any = None
    target_value: Any = None
    delta: float | None = None
    tolerance: float = 0.0
    detail: str = ""
    breakdown: pd.DataFrame | None = field(default=None, repr=False)

    def to_row(self) -> dict:
        return {
            "check": self.name,
            "type": self.check_type.value,
            "status": self.status.value,
            "source": self.source_value,
            "target": self.target_value,
            "delta": self.delta,
            "tolerance": self.tolerance,
            "detail": self.detail,
        }


class ReconciliationEngine:
    def __init__(self, source_url: str, target_url: str):
        self.source: Engine = create_engine(source_url)
        self.target: Engine = create_engine(target_url)
        self.results: list[CheckResult] = []

    # ---- low-level helpers -------------------------------------------------

    def _scalar(self, engine: Engine, sql: str) -> Any:
        with engine.connect() as conn:
            return conn.execute(text(sql)).scalar()

    def _frame(self, engine: Engine, sql: str) -> pd.DataFrame:
        with engine.connect() as conn:
            return pd.read_sql(text(sql), conn)

    @staticmethod
    def _within_tolerance(src: float, tgt: float, tol: float) -> bool:
        if src is None or tgt is None:
            return src == tgt
        if tol == 0:
            return src == tgt
        # tol is a fraction: |delta| / max(|src|,1) <= tol
        denom = max(abs(src), 1.0)
        return abs(src - tgt) / denom <= tol

    # ---- check implementations --------------------------------------------

    def _run_row_count(self, c: dict) -> CheckResult:
        s_sql = f"select count(*) from {c['source_table']}"
        t_sql = f"select count(*) from {c['target_table']}"
        if c.get("where"):
            s_sql += f" where {c['where']}"
            t_sql += f" where {c['where']}"
        s, t = self._scalar(self.source, s_sql), self._scalar(self.target, t_sql)
        ok = s == t
        return CheckResult(
            name=c["name"], check_type=CheckType.ROW_COUNT,
            status=Status.PASS if ok else Status.FAIL,
            source_value=s, target_value=t, delta=(t - s) if s is not None else None,
            detail="exact row count match required",
        )

    def _run_aggregate(self, c: dict) -> CheckResult:
        agg, col = c["agg"], c["column"]
        tol = float(c.get("tolerance", 0.0))
        s = self._scalar(self.source, f"select {agg}({col}) from {c['source_table']}")
        t = self._scalar(self.target, f"select {agg}({col}) from {c['target_table']}")
        s = float(s) if s is not None else None
        t = float(t) if t is not None else None
        ok = self._within_tolerance(s, t, tol)
        return CheckResult(
            name=c["name"], check_type=CheckType.AGGREGATE,
            status=Status.PASS if ok else Status.FAIL,
            source_value=s, target_value=t,
            delta=(t - s) if (s is not None and t is not None) else None,
            tolerance=tol, detail=f"{agg}({col})",
        )

    def _run_distinct_count(self, c: dict) -> CheckResult:
        col = c["column"]
        s = self._scalar(self.source, f"select count(distinct {col}) from {c['source_table']}")
        t = self._scalar(self.target, f"select count(distinct {col}) from {c['target_table']}")
        ok = s == t
        return CheckResult(
            name=c["name"], check_type=CheckType.DISTINCT_COUNT,
            status=Status.PASS if ok else Status.FAIL,
            source_value=s, target_value=t, delta=(t - s),
            detail=f"distinct {col}",
        )

    def _run_null_rate(self, c: dict) -> CheckResult:
        col = c["column"]
        tol = float(c.get("tolerance", 0.0))
        def rate(engine, tbl):
            n = self._scalar(engine, f"select count(*) from {tbl}")
            nulls = self._scalar(engine, f"select count(*) from {tbl} where {col} is null")
            return (nulls / n) if n else 0.0
        s, t = rate(self.source, c["source_table"]), rate(self.target, c["target_table"])
        ok = self._within_tolerance(s, t, tol)
        return CheckResult(
            name=c["name"], check_type=CheckType.NULL_RATE,
            status=Status.PASS if ok else Status.FAIL,
            source_value=round(s, 6), target_value=round(t, 6),
            delta=round(t - s, 6), tolerance=tol, detail=f"null rate of {col}",
        )

    def _run_grouped_count(self, c: dict) -> CheckResult:
        """Counts grouped by key. Surfaces WHICH groups drifted, not just that
        totals differ. This is the check that earns the consulting fee."""
        key = c["group_by"]
        s = self._frame(self.source,
                        f"select {key} as k, count(*) as n from {c['source_table']} group by {key}")
        t = self._frame(self.target,
                        f"select {key} as k, count(*) as n from {c['target_table']} group by {key}")
        merged = s.merge(t, on="k", how="outer", suffixes=("_src", "_tgt")).fillna(0)
        merged["delta"] = merged["n_tgt"] - merged["n_src"]
        drift = merged[merged["delta"] != 0].sort_values("delta", key=abs, ascending=False)
        ok = drift.empty
        return CheckResult(
            name=c["name"], check_type=CheckType.GROUPED_COUNT,
            status=Status.PASS if ok else Status.FAIL,
            source_value=int(s["n"].sum()), target_value=int(t["n"].sum()),
            delta=int(merged["delta"].sum()),
            detail=f"{len(drift)} group(s) drifted on {key}",
            breakdown=drift if not ok else None,
        )

    def _run_referential(self, c: dict) -> CheckResult:
        """Orphan check on the TARGET: child rows whose FK has no parent.
        Run after migration to prove integrity survived the move."""
        sql = (f"select count(*) from {c['child_table']} ch "
               f"left join {c['parent_table']} p on ch.{c['child_key']} = p.{c['parent_key']} "
               f"where p.{c['parent_key']} is null and ch.{c['child_key']} is not null")
        orphans = self._scalar(self.target, sql)
        ok = orphans == 0
        return CheckResult(
            name=c["name"], check_type=CheckType.REFERENTIAL,
            status=Status.PASS if ok else Status.FAIL,
            source_value=0, target_value=orphans, delta=orphans,
            detail=f"orphans in {c['child_table']}.{c['child_key']} → {c['parent_table']}",
        )

    def _run_hash_total(self, c: dict) -> CheckResult:
        """Order-independent row checksum. Each side hashes the concatenated key
        columns per row, casts to a big int, and sums. Equal sums => same row
        contents regardless of ordering. Dialect-specific hashing handled here."""
        cols = " || '|' || ".join(f"cast({col} as varchar)" for col in c["columns"])
        def total(engine, tbl, dialect):
            if dialect == "duckdb":
                sql = f"select sum(hash({cols})::hugeint % 1000000007) from {tbl}"
            elif dialect == "postgresql":
                sql = (f"select sum(('x' || substr(md5({cols}),1,15))::bit(60)::bigint "
                       f"% 1000000007) from {tbl}")
            else:
                sql = f"select sum(abs(hashtext({cols})) % 1000000007) from {tbl}"
            return self._scalar(engine, sql)
        s = total(self.source, c["source_table"], self.source.dialect.name)
        t = total(self.target, c["target_table"], self.target.dialect.name)
        ok = s == t
        return CheckResult(
            name=c["name"], check_type=CheckType.HASH_TOTAL,
            status=Status.PASS if ok else Status.FAIL,
            source_value=s, target_value=t,
            delta=(t - s) if (s is not None and t is not None) else None,
            detail=f"row-content checksum over {c['columns']}",
        )

    # ---- orchestration -----------------------------------------------------

    _DISPATCH = {
        CheckType.ROW_COUNT: "_run_row_count",
        CheckType.AGGREGATE: "_run_aggregate",
        CheckType.DISTINCT_COUNT: "_run_distinct_count",
        CheckType.NULL_RATE: "_run_null_rate",
        CheckType.GROUPED_COUNT: "_run_grouped_count",
        CheckType.REFERENTIAL: "_run_referential",
        CheckType.HASH_TOTAL: "_run_hash_total",
    }

    def run(self, checks: list[dict]) -> list[CheckResult]:
        self.results = []
        for c in checks:
            ctype = CheckType(c["type"])
            method = getattr(self, self._DISPATCH[ctype])
            try:
                self.results.append(method(c))
            except Exception as e:  # an erroring check is a finding, not a crash
                self.results.append(CheckResult(
                    name=c.get("name", "<unnamed>"), check_type=ctype,
                    status=Status.ERROR, detail=f"{type(e).__name__}: {e}",
                ))
        return self.results

    def summary(self) -> dict:
        counts = {s: 0 for s in Status}
        for r in self.results:
            counts[r.status] += 1
        return {
            "generated_at": datetime.now(timezone.utc).isoformat(),
            "total": len(self.results),
            "passed": counts[Status.PASS],
            "failed": counts[Status.FAIL],
            "errored": counts[Status.ERROR],
            "overall": "PASS" if counts[Status.FAIL] == 0 and counts[Status.ERROR] == 0 else "FAIL",
        }

"""
Report rendering for the reconciliation harness.

Three outputs, each with a purpose:
  - console  : fast feedback while iterating
  - markdown : the audit artifact you attach to a cutover sign-off / hand to a
               client PM. This is the deliverable.
  - json     : machine-readable, for CI gates or a dashboard feed.
"""

from __future__ import annotations

import json
from pathlib import Path

from .engine import CheckResult, Status


def _badge(status: Status) -> str:
    return {"PASS": "✓ PASS", "FAIL": "✗ FAIL", "ERROR": "! ERROR"}[status.value]


def to_console(results: list[CheckResult], summary: dict) -> str:
    lines = ["", "RECONCILIATION REPORT", "=" * 60]
    for r in results:
        lines.append(f"{_badge(r.status):10} {r.name}")
        if r.status != Status.PASS:
            lines.append(f"            src={r.source_value} tgt={r.target_value} "
                         f"delta={r.delta} tol={r.tolerance}")
            if r.detail:
                lines.append(f"            {r.detail}")
            if r.breakdown is not None and not r.breakdown.empty:
                head = r.breakdown.head(10).to_string(index=False)
                lines.append("            top drift:\n" +
                             "\n".join("              " + ln for ln in head.splitlines()))
    lines.append("=" * 60)
    lines.append(f"OVERALL: {summary['overall']}  "
                 f"({summary['passed']} passed, {summary['failed']} failed, "
                 f"{summary['errored']} errored)")
    return "\n".join(lines)


def to_markdown(results: list[CheckResult], summary: dict, title: str = "Migration Reconciliation") -> str:
    md = [f"# {title}", "",
          f"**Generated:** {summary['generated_at']}  ",
          f"**Result:** {'✅ PASS' if summary['overall'] == 'PASS' else '❌ FAIL'} "
          f"— {summary['passed']}/{summary['total']} checks passed", "",
          "| Check | Type | Status | Source | Target | Delta | Tol |",
          "|---|---|:---:|--:|--:|--:|--:|"]
    for r in results:
        icon = {"PASS": "✅", "FAIL": "❌", "ERROR": "⚠️"}[r.status.value]
        md.append(f"| {r.name} | {r.check_type.value} | {icon} | "
                  f"{r.source_value} | {r.target_value} | {r.delta} | {r.tolerance} |")
    # drift detail blocks
    drifted = [r for r in results if r.breakdown is not None and not r.breakdown.empty]
    if drifted:
        md += ["", "## Drift detail", ""]
        for r in drifted:
            md += [f"### {r.name}", "", r.breakdown.head(25).to_markdown(index=False), ""]
    errored = [r for r in results if r.status == Status.ERROR]
    if errored:
        md += ["", "## Errors", ""]
        for r in errored:
            md.append(f"- **{r.name}**: {r.detail}")
    return "\n".join(md)


def to_json(results: list[CheckResult], summary: dict) -> str:
    return json.dumps({"summary": summary, "checks": [r.to_row() for r in results]},
                      indent=2, default=str)


def write_all(results, summary, out_dir: str, stem: str = "reconciliation") -> None:
    out = Path(out_dir)
    out.mkdir(parents=True, exist_ok=True)
    (out / f"{stem}.md").write_text(to_markdown(results, summary))
    (out / f"{stem}.json").write_text(to_json(results, summary))
    print(to_console(results, summary))
    print(f"\nWrote {out / stem}.md and {out / stem}.json")

"""
CLI for the reconciliation harness.

Usage:
    uv run reconcile run --config checks.yml --out ./reports
    uv run reconcile run --config checks.yml --fail-on-error   # CI gate

The config file holds connections + checks. Connection URLs can use ${ENV_VAR}
so you never commit credentials.
"""

from __future__ import annotations

import argparse
import os
import re
import sys
from pathlib import Path

import yaml

from .engine import ReconciliationEngine, Status
from .report import write_all

_ENV_PAT = re.compile(r"\$\{([^}]+)\}")


def _expand_env(value: str) -> str:
    def sub(m):
        var = m.group(1)
        if var not in os.environ:
            raise SystemExit(f"Env var {var} referenced in config but not set")
        return os.environ[var]
    return _ENV_PAT.sub(sub, value)


def load_config(path: str) -> dict:
    cfg = yaml.safe_load(Path(path).read_text())
    cfg["source_url"] = _expand_env(cfg["source_url"])
    cfg["target_url"] = _expand_env(cfg["target_url"])
    return cfg


def cmd_run(args: argparse.Namespace) -> int:
    cfg = load_config(args.config)
    engine = ReconciliationEngine(cfg["source_url"], cfg["target_url"])
    results = engine.run(cfg["checks"])
    summary = engine.summary()
    write_all(results, summary, args.out, stem=cfg.get("name", "reconciliation"))

    if summary["failed"] or summary["errored"]:
        return 1 if args.fail_on_error or args.strict else 0
    return 0


def main(argv: list[str] | None = None) -> int:
    p = argparse.ArgumentParser(prog="reconcile",
                                description="Eigenbasis migration reconciliation harness")
    sub = p.add_subparsers(dest="cmd", required=True)
    r = sub.add_parser("run", help="Run checks from a config file")
    r.add_argument("--config", required=True)
    r.add_argument("--out", default="./reports")
    r.add_argument("--fail-on-error", action="store_true",
                   help="Exit non-zero if any check fails (use in CI)")
    r.add_argument("--strict", action="store_true", help="Alias for --fail-on-error")
    r.set_defaults(func=cmd_run)
    args = p.parse_args(argv)
    return args.func(args)


if __name__ == "__main__":
    sys.exit(main())

"""
Self-test for the reconciliation harness.

Builds two DuckDB databases — a 'source' and a deliberately-corrupted 'target' —
then runs the engine and asserts it CATCHES the drift. This is both a unit test
and a live demo you can run in an interview: 'here's the harness catching a
broken migration in real time.'

Run:  uv run python -m tests.test_harness   (from the harness/ dir)
"""

import duckdb

from reconcile import ReconciliationEngine, Status
from reconcile.report import to_console


def build_fixtures(src_path: str, tgt_path: str) -> None:
    # SOURCE: the truth
    s = duckdb.connect(src_path)
    s.execute("""
        create or replace table orders as
        select * from (values
            ('o1','c1','delivered', 100.0),
            ('o2','c2','delivered', 200.0),
            ('o3','c3','shipped',   150.0),
            ('o4','c4','canceled',   50.0),
            ('o5','c5','delivered', 300.0)
        ) t(order_id, customer_id, order_status, price)
    """)
    s.close()

    # TARGET: a migration with THREE planted defects:
    #   1. o5 dropped entirely        -> row_count + price_sum + grouped_count fail
    #   2. o3 status flipped to delivered -> grouped_count + hash fail
    #   3. (counts of delivered now differ)
    t = duckdb.connect(tgt_path)
    t.execute("""
        create or replace table orders as
        select * from (values
            ('o1','c1','delivered', 100.0),
            ('o2','c2','delivered', 200.0),
            ('o3','c3','delivered', 150.0),
            ('o4','c4','canceled',   50.0)
        ) t(order_id, customer_id, order_status, price)
    """)
    t.close()


def main():
    src, tgt = "/tmp/recon_src.duckdb", "/tmp/recon_tgt.duckdb"
    build_fixtures(src, tgt)

    checks = [
        {"name": "orders_row_count", "type": "row_count",
         "source_table": "orders", "target_table": "orders"},
        {"name": "price_sum", "type": "aggregate", "agg": "sum", "column": "price",
         "source_table": "orders", "target_table": "orders", "tolerance": 0.0},
        {"name": "distinct_customers", "type": "distinct_count", "column": "customer_id",
         "source_table": "orders", "target_table": "orders"},
        {"name": "orders_by_status", "type": "grouped_count", "group_by": "order_status",
         "source_table": "orders", "target_table": "orders"},
        {"name": "content_hash", "type": "hash_total",
         "columns": ["order_id", "customer_id", "order_status"],
         "source_table": "orders", "target_table": "orders"},
