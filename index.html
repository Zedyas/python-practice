Write a DataFrame validator

Medium

Practice task: load the quality-check fixture and write a validator for the DataFrame. It should confirm required columns exist, sku values are unique, and price is never negative. Return a list of readable issue messages; return an empty list when the data passes.

import pandas as pd

df = pd.read_csv('data/parts_quality.csv')
Hint #1

Return a report of all issues that can be checked safely; only stop early if required columns are missing.

Hint #2

Load data/parts_quality.csv, then use df.columns, df['sku'].duplicated(), and boolean checks like (df['price'] < 0).sum().

Solution
import pandas as pd

df = pd.read_csv('data/parts_quality.csv')

def validate_parts(df, required=('sku', 'name', 'price')):
    issues = []

    missing = [c for c in required if c not in df.columns]
    if missing:
        issues.append(f'missing columns: {missing}')
        return issues  # later checks depend on these columns

    dupes = df['sku'][df['sku'].duplicated()].unique()
    if len(dupes):
        issues.append(f'{len(dupes)} duplicate skus, e.g. {dupes[:3].tolist()}')

    bad_price = (df['price'] < 0).sum()
    if bad_price:
        issues.append(f'{bad_price} rows with negative price')

    return issues

issues = validate_parts(df)

# issues → ["1 duplicate skus, e.g. ['CMP-100']", '1 rows with negative price']
# (empty list [] would mean the data passed every check)

Why it matters: returning a list of issues (instead of raising on the first) gives a usable data-quality report — exactly what you'd log before a pipeline publishes output.

Safe nested lookup by dotted path

Medium

Practice task: write get_nested(data, path, default=None). It should walk a dotted path such as 'usage.tokens.total' through nested dictionaries and return default if any key is missing or any intermediate value is not a dict.

Hint #1

Walk one path segment at a time and bail out as soon as the current value stops being a dict.

Hint #2

Expected: For usage.tokens.total, the examples return 500 and then 0 for the missing path default.

Solution
data = {'usage': {'tokens': {'total': 500}}, 'model': 'flash'}

def get_nested(data, path, default=None):
    current = data
    for key in path.split('.'):
        if not isinstance(current, dict) or key not in current:
            return default
        current = current[key]
    return current

get_nested(data, 'usage.tokens.total')      # 500
get_nested(data, 'usage.cost.usd', 0)       # 0

Why it matters: chained data['a']['b']['c'] is the #1 source of KeyError/TypeError crashes in connectors fed by third-party APIs.

Flatten nested JSON to dotted keys

Medium

Practice task: write a function that flattens nested dictionaries into one dictionary with dotted keys. Keep non-dict values as the final values.

# becomes
{'model': 'gemini', 'limits.rpm': 60, 'limits.retry.max': 3}
Hint #1

This is a recursive traversal: carry the current prefix down into nested dictionaries.

Hint #2

Use out.update(flatten(value, f'{full_key}.')) when a value is another dict.

Expected: The sample becomes {"model": "gemini", "limits.rpm": 60, "limits.retry.max": 3}.

Solution
def flatten(d, prefix=''):
    out = {}
    for key, value in d.items():
        full_key = f'{prefix}{key}'
        if isinstance(value, dict):
            out.update(flatten(value, f'{full_key}.'))
        else:
            out[full_key] = value
    return out

Follow-up they may ask: what about lists? Common answers: keep them as-is, JSON-encode them, or index into the key (items.0.sku) — depends on the destination schema.

    match = re.fullmatch(r'\s*([\d.]+)\s*(ms|s|m)\s*', str(text))
    if not match:
        return None
    value, unit = match.groups()
    try:
        return float(value) * UNIT_TO_SECONDS[unit]
    except ValueError:
        return None  # e.g. '1.2.3s'

Details: put the longest unit first in the alternation (ms|s|m) so it wins, and use fullmatch (not match) so trailing garbage like '2m later' doesn't sneak through. The inner try catches malformed numbers like '1.2.3' that the character class allows.

import functools
import time

class RetryableError(Exception):
    pass

Hint #2

Solution
import time

class RetryableError(Exception):
    pass

def call_with_retries(fn, *, deadline_s=30, max_attempts=5, base_delay=1.0):
    start = time.monotonic()
    last_error = None
    for attempt in range(1, max_attempts + 1):
        try:
            return fn()
        except RetryableError as e:
            last_error = e
            remaining = deadline_s - (time.monotonic() - start)
            if attempt == max_attempts or remaining <= 0:
                raise
            delay = min(base_delay * 2 ** (attempt - 1), remaining)
            time.sleep(delay)
    raise last_error

Why it matters: per-call retries without an overall deadline are how background jobs hang. Deadlines make failure bounded and observable.

Retries with backoff and jitter

Medium

Hint #1

Hint #2

Catch timeout/connection/retry errors; use base_delay * 2 ** (attempt - 1) + random.uniform(0, 1).

Solution
import random
import time
import requests

def fetch_with_retries(url, max_attempts=4, base_delay=1.0):
    for attempt in range(1, max_attempts + 1):
        try:
            resp = requests.get(url, timeout=10)
            if resp.status_code >= 500 or resp.status_code == 429:
                raise requests.exceptions.RetryError(
                    f'HTTP {resp.status_code}'
                )
            resp.raise_for_status()      # non-retryable 4xx → raises here
            return resp.json()
        except (requests.exceptions.Timeout,
                requests.exceptions.ConnectionError,
                requests.exceptions.RetryError):
            if attempt == max_attempts:
                raise
            delay = base_delay * 2 ** (attempt - 1) + random.uniform(0, 1)
            time.sleep(delay)

Why jitter: if 50 workers all fail at once and back off identically, they all retry at the same instant and knock the service over again (the "thundering herd"). Randomness spreads them out. Knowing why 4xx shouldn't be retried is the other half of this question.

import time
import requests

    product_ids = json.load(f)['product_ids']

    def __init__(self, per_second):
        self.min_interval = 1.0 / per_second
        self._last_call = 0.0
